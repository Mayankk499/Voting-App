voting app functionality

1. user sign in/ sign up
2. see the list of candidate
3. vote one of the candidate
4. there is a route which shows the list of candidate and their live vote counts sorted by their vote count
5. user data must contain their one unique goverment id proof named: aadhar card number
6. there should be one admin who can only maintain the table of candidates and he can't able to vote at all
7. user can change their password
8. user can only login with there aadhar card number & password
9. admin can't vote at all

Routes

User Authentication:
    /signup: POST - Create a new user account.
    /login: POST - Log in to an existing account. [aadhar card number + password]

Voting:
    /candidates: GET - get the list of candidates.
    /vote/:candidateId: POST - vote for a specific candidate.

Vote Counts:
    /vote/counts: GET - get the list of candidates sorted by their vote counts

User Profile:
    /profile: GET - get the user's profile information.
    /profile/password: PUT - Chnage the user's password.
    
Admin Candidate Management:
    /candidates: POST - Create a new candidate.
    /candidate/:candidateId: PUT - update an existing candidate.
    /candidate/:candidateId: DELETE - Delete a candidate from the list.
